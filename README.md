# 暗号資産スナップショット（ローカル版）

DeBankで手動コピーしたHTMLを、ブラウザ上のローカル画面から取り込み、アドレス別・資産別・DeFi別に保存・表示します。

起動直後の「資産概要」では、全アドレスの合算総額、資産推移、資産別配分、実体資産ごとの数量・評価額、保管場所を確認できます。LidoやAaveは資産名ではなくプロトコル（保管・運用場所）として扱い、stETHやUSDTなどの実体資産を表示します。

## 起動

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

Braveで <http://127.0.0.1:8877> を開きます。

## 複数アドレス

1. 「データ更新・設定」でアドレス1〜5を登録して保存します。
2. 更新対象を選び、「DeBankで開く」で対象アドレスを開きます。
3. 利用者がブラウザ拡張でHTMLをコピーします。
4. 「クリップボードを解析」を押し、アドレス・総額・検出件数を確認します。
5. 「このスナップショットを保存」を押して確定します。
6. アドレスごとに1〜5を繰り返します。

ダッシュボードは各アドレスの最新スナップショットを一度だけ合算するため、アドレス1〜5の合計を二重計上しません。解析のみではデータを保存せず、保存ボタンで確定した時だけJSONLに追記します。

## 基準日

画面の「記録日」は、スナップショットを整理するために利用者が付ける日付ラベルです。実際にHTMLをコピー・取り込んだ時刻は `captured_at` として別にUTC保存します。アプリがDeBankから指定した過去時点の残高を取得する機能ではありません。

## 表示ルール

- 総資産にはすべての評価額を含めます。
- 資産一覧と円グラフでは、評価額が1 USD未満の資産を初期状態で隠します。必要なら「$1未満も表示」で確認できます。
- 資産別集計は、直持ちとDeFi内ポジションを資産記号ごとにまとめます。
- DeFiの資産記号が解析できない場合は、プロトコル名で代用せず「資産不明」として警告対象にします。

## データ

`data/` に次のファイルが作成されます。

- `wallets.json`: 登録アドレス
- `snapshots.jsonl`: アドレスごとの取得結果（追記型）
- `runs.jsonl`: 取得実行の履歴

`data/` はGit管理対象外です。HTML原本は保存せず、入力HTMLのSHA-256だけをスナップショットへ保存します。

## GitHub公開時の注意

このリポジトリにはアプリ本体だけを含め、実ウォレットのアドレス、残高履歴、DeBankからコピーしたHTMLは含めません。

`.gitignore` では `data/`、`*.jsonl`、`sample-*.html`、`*-debank.html` を除外しています。公開前に `git status --short` で、個人用データがステージされていないことを確認してください。
