# 暗号資産ポートフォリオ（ローカル版）

EVMウォレットと暗号資産取引所の残高をローカルで取得・保存し、資産別・保管場所別に統合表示する個人向けWebアプリです。外部ホスティングやデータベースを必要とせず、取得結果はローカルのJSONLへ保存します。

## 主な機能

- 複数ウォレットのDeBank表示内容をスナップショットとして保存
- Binance Spot、Bybit UTA、bitFlyer、Coincheck、bitbank、GMOコインの読取専用残高API
- ウォレット、DeFi、取引所を横断した総資産・資産数量・評価額の合算
- 資産推移、トークン別配分、保管場所別内訳
- USD/JPYレートを取得時点で保存し、USD・JPYを併記
- stETH、USDTなどの実体資産と、Lido、Aaveなどのプロトコルを分離
- 利用可能残高、注文拘束残高、借入・負債の正規化
- 1 USD未満の資産を保持したまま、標準画面では非表示
- デジタル庁デザインシステムの考え方を踏襲したローカルダッシュボード

## 起動

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

DeBankの全ウォレット一括取得を利用する場合は、初回だけPlaywright用ブラウザを追加します。

```bash
playwright install chromium
```

ブラウザで <http://127.0.0.1:8877> を開きます。サーバーは `127.0.0.1` のみに公開されます。別ポートを使う場合は `PORT=8878 python app.py` のように指定できます。

すでに起動済みの場合は、一度停止してから再起動してください。取引所追加は「設定」から行い、取引・送金・出金権限を持たない読取専用APIキーだけを使用してください。

## ウォレットデータの更新

### 手動HTML取込

1. 「設定」で公開アドレスを登録します。
2. 更新対象を選び、「DeBankで開く」で対象アドレスを開きます。
3. 利用者がブラウザ拡張でHTMLをコピーします。
4. 「クリップボードを解析」を押し、アドレス・総額・検出件数を確認します。
5. 「このスナップショットを保存」を押して確定します。
6. 必要なアドレスについて繰り返します。

解析のみではデータを保存せず、保存操作で確定した時だけJSONLへ追記します。

### 全ウォレット一括取得（任意）

「全ウォレットを自動取得」は、登録済みの有効ウォレットをPlaywrightのChromiumで順番に開き、表示完了後のHTMLを解析して同じ更新セッションとして保存します。

- ウォレットは並列巡回せず、1件ずつ取得します。
- ウォレットごとにタブを閉じ、次の取得までランダムな待機時間を設けます。
- 1件の失敗で全体を中断せず、成功・失敗をウォレット単位で記録します。
- DeBankの画面構造やアクセス条件の変更により失敗する可能性があります。
- 自動取得が利用できない場合も、手動HTML取込を利用できます。

自動取得は登録済みの有効ウォレットを1件ずつ処理し、取得結果を各スナップショットへ保存します。自動取得できない場合は手動HTML取込を利用できます。

## 取引所データの更新

1. 「設定」で取引所と表示名を追加します。
2. 読取専用のAPI KeyとSecretを登録します。
3. 接続テストで残高参照を確認します。
4. 「データ更新」で残高を取得し、プレビューを確認します。
5. 問題がなければスナップショットを保存します。

API KeyとSecretはmacOS Keychainへ保存し、設定JSON、スナップショット、ログ、Git管理対象には保存しません。出金権限付きのAPIキーは使用しないでください。

## 基準日

ウォレット画面の「記録日」は、スナップショットを整理するための日付ラベルです。実際の取得時刻は `captured_at` としてUTC保存します。指定した過去時点の残高をDeBankや取引所から取得する機能ではありません。

## 表示ルール

- ダッシュボードは各ウォレット・取引所の最新スナップショットを1件ずつ採用します。
- 総資産には取得元が返した総額を使用し、解析できた資産内訳との差を検証します。
- 資産一覧と円グラフでは、評価額が1 USD未満の資産を初期状態で隠します。必要なら「$1未満も表示」で確認できます。
- 資産別集計は、ウォレット直持ち、DeFi、取引所の同一資産を合算します。
- DeFiの資産記号が解析できない場合は、プロトコル名で代用せず「資産不明」として警告対象にします。
- 推移グラフは保存済みスナップショットだけを使い、同一取得元・同一記録日の重複を除外します。

## データ

`data/` に次のファイルが作成されます。

- `wallets.json`: 登録アドレス
- `sources.json`: 取引所接続設定（秘密情報を除く）
- `snapshots.jsonl`: アドレスごとの取得結果（追記型）
- `portfolio-snapshots.jsonl`: 取引所ごとの取得結果（追記型）
- `runs.jsonl`: 取得実行の履歴

`data/` はGit管理対象外です。DeBank HTML原本は保存せず、入力HTMLのSHA-256だけをスナップショットへ保存します。

## テスト

```bash
python3 -m unittest discover -s tests -v
python3 -m py_compile app.py exchange.py debank_auto.py
```

取引所コネクタのテストでは、外部APIをモックし、利用可能残高・拘束残高・借入の正規化を検証します。

## GitHub公開時の注意

このリポジトリにはアプリ本体だけを含め、実ウォレットのアドレス、残高履歴、DeBankからコピーしたHTMLは含めません。

`.gitignore` では `data/`、`*.jsonl`、`sample-*.html`、`*-debank.html` を除外しています。公開前に `git status --short` で、個人用データがステージされていないことを確認してください。
