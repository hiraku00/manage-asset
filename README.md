# 暗号資産ポートフォリオ（ローカル版）

EVMウォレットと暗号資産取引所の残高をローカルで取得・保存し、資産別・保管場所別に統合表示する個人向けWebアプリです。外部ホスティングやデータベースを必要とせず、取得結果はローカルのJSONLへ保存します。

## 主な機能

- 複数ウォレットのDeBank表示内容をスナップショットとして保存
- Binance、Bybit、bitFlyer、Coincheck、bitbank、GMOコインの読取専用残高API
- ウォレット、DeFi、取引所を横断した総資産・資産数量・評価額の合算
- 資産推移、トークン別配分、保管場所別内訳
- 通貨ごとの残高・日次増加量・資産推移・評価額の履歴表示
- 推移グラフの7日・30日・90日・全期間切り替えと、履歴表のページング
- USD/JPYレートを取得時点で保存し、USD・JPYを併記
- JPY残高はUSD/JPYレートでUSD評価し、Binanceの暗号資産ティッカーには問い合わせない
- stETH、USDTなどの実体資産と、Lido、Aaveなどのプロトコルを分離
- 利用可能残高、注文拘束残高、借入・負債の正規化
- 1 USD未満の資産を保持したまま、標準画面では非表示
- Appleのインターフェース設計原則をWebへ翻訳した、流動的でアクセシブルなローカルダッシュボード

## 起動

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

DeBankの全ウォレット一括取得を利用する場合は、初回だけPlaywright用ブラウザを追加します。

```bash
playwright install chromium
```

ブラウザで <http://127.0.0.1:8877> を開きます。サーバーは `127.0.0.1` のみに公開されます。別ポートを使う場合は `PORT=8878 python app.py` のように指定できます。

すでに起動済みの場合は、一度停止してから再起動してください。取引所追加は「設定」から行い、取引・送金・出金権限を持たない読取専用APIキーだけを使用してください。

## ウォレットデータの更新

### 個別更新

1. 「設定」で公開アドレスを登録します。
2. 「データ更新」で更新対象ウォレットを選択します。
3. 「更新」を押すと、選択したウォレットをヘッドレスブラウザで取得・保存します。

### 全ウォレット一括取得（任意）

「全ウォレットを更新」は、登録済みの有効ウォレットをPlaywrightのChromiumで順番に開き、表示完了後のHTMLを解析して同じ更新セッションとして保存します。

- ウォレットは並列巡回せず、1件ずつ取得します。
- ウォレットごとにブラウザを閉じます。次のウォレット開始前に追加の待機時間は設けません。
- 1件の失敗で全体を中断せず、成功・失敗をウォレット単位で記録します。
- 実行中は対象件数、完了件数、成功・失敗件数、経過秒数を画面に表示します。
- 取得完了後は各ウォレットのスナップショットを自動保存します。
- DeBankの画面構造やアクセス条件の変更により失敗する可能性があります。

## 取引所データの更新

1. 「設定」で取引所と表示名を追加します。
2. 読取専用のAPI KeyとSecretを登録します。
3. 接続テストで残高参照を確認します。
4. 「データ更新」で接続先を選択し、「更新」を押します。
5. 複数の取引所をまとめて取得する場合は「全取引所を更新」を押します。

接続テストはAPI参照の確認のみで、残高を保存しません。保管場所や総資産へ反映するには、必ず「データ更新」から残高更新を実行してください。

一括更新中は対象数、完了数、残り、成功数、失敗数、経過秒数を画面に表示します。完了時は成功・失敗件数と処理時間を表示し、失敗した接続先とエラー内容を確認できます。

API KeyとSecretはmacOS Keychainへ保存し、設定JSON、スナップショット、ログ、Git管理対象には保存しません。出金権限付きのAPIキーは使用しないでください。

## 取得日時

取得日時はアプリが自動的に付与し、`captured_at` として保存します。利用者が記録日を入力したり、指定した過去時点の残高をDeBankや取引所から取得したりする機能はありません。

## 表示ルール

- ダッシュボードは各ウォレット・取引所の最新スナップショットを1件ずつ採用します。
- 総資産には取得元が返した総額を使用し、解析できた資産内訳との差を検証します。
- 資産一覧と円グラフでは、評価額が1 USD未満の資産を初期状態で隠します。必要なら「$1未満も表示」で確認できます。
- 資産別集計は、ウォレット直持ち、DeFi、取引所の同一資産を合算します。
- 保有資産には合計数量、単価、評価額、比率、主な保管場所を表示します。数量は最大4桁の小数で表示します。
- 金額はUSDを小数2桁固定、円換算を小数なしで表示します。
- DeFiの資産記号が解析できない場合は、プロトコル名で代用せず「資産不明」として警告対象にします。
- 推移グラフは保存済みスナップショットだけを使い、同一取得元・同一記録日の重複を除外します。
- 資産概要の前日保存比は、前日（保存がなければ直近の過去日）の最初の保存値と最新値を比較します。
- 通貨推移の日次増加量は、原則として前の記録日の残高との差分です。取得を休んだ場合は、直前に保存された記録との差分が次の記録日に計上され、上部カードには選択期間の合計を表示します。
- 通貨推移の資産推移モードでは、選択期間の残高推移と、表の `Change (USD/JPY)` の期間合計を表示します。
- stETHの期間報酬はLido履歴の `type=reward` だけを合計し、TransferやStaking入金を除外します。Lido CSVの最終日より後だけは、次回CSV取込までスナップショット残高差から暫定計算します。
- 通貨推移の表はグラフ期間とは独立し、日付の降順で1ページ10件を表示します。
- JPYは取得時点のUSD/JPYレートで `JPY数量 ÷ USD/JPY` としてUSD評価し、総資産へ合算します。円表示は併記します。
- 価格取得できない資産も数量は保存しますが、評価額は合算せず「評価なし」と表示します。

## データ

`data/` に次のファイルが作成されます。

- `wallets.json`: 登録アドレス
- `sources.json`: 取引所接続設定（秘密情報を除く）
- `snapshots.jsonl`: アドレスごとの取得結果（追記型）
- `portfolio-snapshots.jsonl`: 取引所ごとの取得結果（追記型）
- `runs.jsonl`: 取得実行の履歴
- `lido-rewards.jsonl`: Lido CSVから移行した個人のReward履歴（任意）
- `usd-jpy-rates.json`: 履歴表示に用いる日次USD/JPYレート

残高・取引履歴などの個人データを含む `data/` は原則としてGit管理対象外です。例外として、公開市場データである `data/usd-jpy-rates.json` だけを管理対象にしています。DeBank HTML原本は保存せず、入力HTMLのSHA-256だけをスナップショットへ保存します。

### Lido履歴の移行

Lidoから取得したCSVをローカルのJSONLへ変換する場合は、次を実行します。CSVと変換後のJSONLはいずれもGit管理対象外です。

```bash
python3 scripts/migrate_lido_rewards.py "/path/to/Lido Rewards.csv"
```

変換結果は既定で `data/lido-rewards.jsonl` に保存され、以後のスナップショット履歴と画面表示時に統合されます。CSV原本をリポジトリ内へコピーする必要はありません。

## テスト

```bash
pytest -q
node --test tests/portfolio-core.test.js
python3 -m py_compile app.py exchange.py debank_auto.py
node --check static/portfolio-core.js
node --check static/app-ui.js
```

取引所コネクタのテストでは外部APIをモックし、利用可能残高・拘束残高・借入の正規化を検証します。ポートフォリオ集計テストではDeFi解析、最新スナップショット選択、単価計算、未評価資産を検証します。Playwright回帰テストでは主要5画面の分離、通貨推移、ページング、更新カード、各画面の表示崩れを確認します。テストは個人のLidoデータに依存せず、合成データを使用します。

## フロントエンド構成

- `static/index.html`: 主要5画面の固定HTML構造
- `static/display-config.js`: 通貨画面の残高・増加量・法定通貨などの表示桁数
- `static/styles.css`: 半透明マテリアル、即時フィードバック、レスポンシブ構成、ダークモードと動きを減らす設定を備えたデザインシステム
- `static/portfolio-core.js`: 集計、DeFi解析、単価、履歴計算の純粋関数
- `static/app-ui.js`: API接続、描画、更新・設定操作

表示桁数は `static/display-config.js` で管理します。`fiatDecimals` のUSD・JPYは全画面へ適用され、`tokenDecimals` では通貨数量の残高（`balance`）と差分・Reward（`change`）を分けて指定できます。必要な場合だけ `currencyOverrides` で通貨別に上書きします。

## GitHub公開時の注意

このリポジトリにはアプリ本体だけを含め、実ウォレットのアドレス、残高履歴、DeBankからコピーしたHTMLは含めません。

`.gitignore` では `data/`、`*.jsonl`、`sample-*.html`、`*-debank.html` を除外し、公開情報の `data/usd-jpy-rates.json` だけを例外にしています。LidoのCSV・JSONL、ウォレット設定、残高スナップショット、実行ログはコミットしません。公開前に `git status --short` と `git diff --cached --name-only` で、個人用データがステージされていないことを確認してください。
