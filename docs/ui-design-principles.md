# UI Design Principles（現行）

- 文書版: 1.0
- 更新日: 2026-07-19
- 対象: `static/index.html`、`static/styles.css`、`static/app-ui.js`
- 位置付け: 現行のビジュアル、テーマ、レスポンシブ、モーション仕様
- 参照: [Apple Design Principles](https://developer.apple.com/design/human-interface-guidelines/design-principles)、[Designing for macOS](https://developer.apple.com/design/human-interface-guidelines/designing-for-macos)

## 方針

Apple公式のDesign PrinciplesとmacOS HIGを、macOS上のブラウザで使うローカルWebアプリへ翻訳する。大きな画面の情報密度、ツールバーの階層、精密なポインタ操作、キーボード操作、リサイズへの適応を重視し、資産を確認する時間を短くする。アクセシビリティと情報設計の要件もこの文書で一元管理する。

- 目的: 総資産、増減、配分、鮮度を最初に理解できる
- 直接性: 押下時に即時フィードバックを返し、画面遷移は現在値から自然に開始する
- 空間的一貫性: ウィンドウクローム、ツールバー、コンテンツの階層を標準マテリアルと余白で表現する
- macOS適応: 広い表示領域では情報を過度に階層化せず、ツールバーとコンテンツを分離する
- Agency: 画面を固定したフローに閉じ込めず、ナビゲーションとキーボードショートカットから各画面へ直接移動できる
- Craft: ナビゲーションに⌥1〜⌥5のショートカットを付け、タイトルツールチップで発見可能にする
- 配慮: `prefers-reduced-motion`、`prefers-reduced-transparency`、`prefers-contrast`を尊重する
- 柔軟性: OSテーマに追従し、ヘッダーからライト／ダークを手動切替できる。選択は`localStorage`へ保存する

## テーマ

通常はOSのカラースキームへ追従する。利用者がヘッダーの切替ボタンを押すと明示選択へ移行し、`portfolio-theme`へ`light`または`dark`を保存する。ライトへ戻した場合も同じ操作で切り替えられる。

ダークモードでは、本文色、補助色、リンク色、主要操作色を同じ青で代用しない。ドーナツ中央の数値やSVG内テキストもCSSテーマトークンから色を受け取り、固定黒・固定灰を残さない。

## レスポンシブ

デスクトップでは概要カードと資産配分を2列で表示する。900px以下ではナビを下部のフローティングバーへ移し、560px以下では保有資産テーブルをカード行へ変換する。情報を隠すのではなく、ラベル付きで再配置する。

## macOSブラウザとしての適用範囲

- ブラウザのウィンドウサイズ変更、フルスクリーン、複数ディスプレイはブラウザに委譲し、レイアウトは利用可能な幅へ適応する。
- ネイティブmacOSのメニューバーやSF Symbolsは使用せず、Web標準のツールバー、ARIAラベル付きSVG、キーボード操作で同じ役割を担保する。
- Webではアプリ固有のテーマ切替を提供するが、初期状態はOSの`prefers-color-scheme`へ追従する。
- カードはコンテンツ面の標準マテリアルとし、Liquid Glass相当の透過表現はウィンドウクロームや操作面に限定する。
