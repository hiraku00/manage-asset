# 暗号資産ポートフォリオ 設計ドキュメント

このディレクトリの入口です。現行実装と異なる過去仕様を正として参照しないよう、文書の責務と優先順位をここで定義します。

## まず読む文書

| 優先 | 文書 | 役割 | 状態 |
|---|---|---|---|
| 1 | [UI Design Principles](ui-design-principles.md) | 画面、テーマ、レスポンシブ、モーション、アクセシビリティ | 現行の正 |
| 2 | [取引所統合・ダッシュボード詳細設計](exchange-integration-design.md) | API、スナップショット、集計、更新、データ保護 | 現行の正 |

READMEの機能一覧、表示ルール、起動方法は実装利用者向けの要約です。設計判断では本ページからリンクされた現行文書を優先します。

## 現行仕様の境界

- UIの見た目と操作: Apple DesignをWebへ翻訳した`ui-design-principles.md`
- 情報設計とアクセシビリティ: 同文書のアクセシビリティ節および実装されたARIA、フォーカス、コントラスト、縮小モーション対応
- 取得・保存・集計: `exchange-integration-design.md`と実装コード
- 機密情報: READMEと取引所統合設計のKeychain／Git管理外ルール

## 削除した文書

次の文書は、実装前の構想または旧画面仕様であり、現行実装へ適用していないため削除しました。

- 銀行・証券まで含む初期提案書
- ウォレット限定MVP設計書
- 旧ダッシュボードUX再設計書
- 実装前の旧UI監査書

旧監査文書を残すと、現在のApple Designベースの見た目と異なる仕様を参照する原因になります。コントラスト、フォーカス、キーボード操作、見出し、フォーム、テーブルなど、今も必要な要件だけを現行UI方針へ統合しています。

## 更新ルール

1. 現行実装へ影響する設計変更は、該当する現行文書とREADMEを同じコミットで更新する。
2. 実装と異なる案は、現行文書へ追記せず、IssueやPRで提案として管理する。
3. 文書を追加するときは、この一覧に「役割」と「状態」を追加する。
